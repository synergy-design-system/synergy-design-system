# Localizing Synergy components

Components can be localized by importing the appropriate translation file and setting the desired [`lang`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Global_attributes/lang) attribute and/or [`dir`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Global_attributes/dir) attribute on the `<html>` element.
Changing the lang attribute will automatically update all localized components to use the new locale.

> Synergy provides a localization mechanism for component internals.
> This is **not** designed to be used as localization tool for your entire application.
> You should use a more appropriate tool if you need to localize content in your app.

## Static examples

Here’s an example that renders Synergy components in Spanish, using plain html.

```html
<html lang="es">
  <head>
    <script type="module" src="/path/to/synergy/dist/translations/es.js"></script>
  </head>
  <body>...</body>
</html>
```

```html
<html>
  <body>...</body>
  <script type="module">
    // Alternative usage via Javascript.
    // Same as setting <html lang="es">
    import '@synergy-design-system/components/translations/es.js';
    document.documentElement.lang = 'es';
  </script>
</html>
```

## Translation resolution

The locale set by `<html lang="...">` is the default locale for the document.
If a country code is provided, e.g. `es-PE` for Peruvian Spanish, the localization library will resolve it like this:

1. Look for `es-PE`
2. Look for `es`
3. Fall back to `en`

Synergy uses English as a fallback to provide a better experience than rendering nothing or throwing an error.

## Multiple locales per page

You can use a different locale for an individual component by setting its `lang` and/or `dir` attributes.
Here is a contrived example to demonstrate.

```html
<html lang="es">
  <body>
    <syn-button><!-- Spanish --></syn-button>
    <syn-button lang="ru"><!-- Russian --></syn-button>
  </body>
</html>
```

For performance reasons, the `lang` and `dir` attributes must be on the component itself, not on an ancestor element.
This limitation exists because there’s no efficient way to determine the current locale of a given element in a DOM tree.

```html
<html lang="es">
  <body>
    <div lang="ru">
      <syn-button><!-- still in Spanish --></syn-button>
    </div>
  </body>
</html>
```

## Available localizations

Synergy ships with a number of translations.
The default is English (US), which also serves as the fallback locale.
As such, you do not need to import the English translation.
To see a list of all available translations in the latest version, [refer to this directory](https://github.com/synergy-design-system/synergy-design-system/tree/main/packages/components/src/translations).

You do not need to load translations up front.
You can import them dynamically even after updating the `lang` attribute.
Once a translation is registered, localized components will update automatically.
